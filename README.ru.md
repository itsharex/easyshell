# EasyShell

**AI-нативная платформа серверных операций**

Пусть ИИ пишет скрипты, управляет задачами на нескольких хостах и анализирует вашу инфраструктуру — пока вы фокусируетесь на главном.

## Почему EasyShell?

Традиционные инструменты управления серверами требуют от вас написания каждого скрипта, подключения по SSH к каждому узлу и самостоятельной интерпретации каждого вывода. EasyShell меняет эту модель: **ИИ выступает оператором, а вы — принимаете решения.**

- **Опишите задачу простым языком** → ИИ напишет готовые к работе shell-скрипты с проверкой изменений
- **Поставьте цель для нескольких хостов** → ИИ спланирует шаги выполнения, запустит их и обобщит результаты
- **Подключайтесь через Web SSH** → Полноценный терминал с файловым менеджером, вкладками и поиском — без локального клиента

## Основные возможности

### 1. AI Script Assistant

> Опишите, что вам нужно. ИИ напишет скрипт. Проверьте изменения. Примените в один клик.

AI Script Workbench — это редактор с разделенной панелью, где вы описываете требования на естественном языке, а ИИ генерирует или изменяет shell-скрипты для выбранной ОС. Написание скрипта отображается в реальном времени. Встроенный diff-режим подсвечивает все изменения. Вкладка сводки объясняет модификации на вашем языке.

</p>

### 2. AI Task Orchestration

> «Проверь диск и память на всех хостах, отметь всё, что выше 80%, и предложи решения.» — Готово.

Интерфейс AI Chat позволяет ставить операционные задачи высокого уровня. ИИ разбивает их на многоэтапный план выполнения (исследование → анализ → отчет), отправляет скрипты на целевые хосты, собирает результаты и предоставляет структурированный анализ с оценкой рисков и практическими рекомендациями — и все это за один цикл общения.

<p align="center">

</p>

### 3. Полнофункциональный Web SSH

> Настоящий терминал. Настоящий файловый менеджер. SSH-клиент не требуется.

Веб-терминал промышленного уровня с поддержкой нескольких сессий, встроенным файловым менеджером (загрузка, скачивание, создание, удаление, навигация), полнотекстовым поиском по буферу терминала и устойчивыми соединениями через WebSocket. Управляйте файлами и запускайте команды в одном окне.

### 4. Управление хостами и мониторинг

> Единый обзор всех серверов с отображением состояния в реальном времени, пакетными операциями и управлением жизненным циклом агентов.

Управляйте хостами индивидуально или массово — добавляйте через форму или импорт CSV, организуйте в кластеры, отслеживайте состояние подключения и разворачивайте/обновляйте агенты одним кликом. Единая панель управления показывает метрики работоспособности.

### 5. Потоковые логи в реальном времени

> Наблюдайте за выполнением скриптов на всех целевых хостах в прямом эфире.

При отправке скрипта EasyShell транслирует вывод от каждого агента в реальном времени. Цветовая маркировка, временные метки и фильтрация по хостам помогают мгновенно выявлять проблемы — больше не нужно ждать завершения пакетных заданий.

### 6. Безопасность и контроль рисков

> Встроенные меры безопасности: workflow согласования, журнал аудита, ограничения операций.

Настраивайте, какие операции требуют одобрения перед выполнением. Все действия журналируются для соответствия требованиям. Ролевое управление доступом ограничивает права пользователей, а чувствительные команды можно помечать или полностью блокировать.

## Автоматизированные проверки на базе ИИ

> **Cron → Скрипт → AI-анализ → Уведомление бота** — полностью автоматизировано, без участия человека.

Планируйте задачи проверки с помощью cron-выражений. EasyShell отправляет скрипты агентам, собирает вывод (состояние диска, памяти, сервисов, логи), передает его вашей ИИ-модели для анализа и отправляет отчет вашей команде через каналы ботов.

**Как это работает:**
1. **Настройка** — cron-выражение + shell-скрипт + промпт для ИИ-анализа
2. **Выполнение** — EasyShell отправляет задачи целевым агентам по расписанию
3. **Анализ** — Вывод отправляется ИИ-модели (OpenAI / Gemini / GitHub Copilot / Custom)
4. **Уведомление** — Отчет отправляется в канал бота (Telegram, Discord, Slack, DingTalk, Feishu, WeCom)

**Поддерживаемые каналы ботов** ([Руководство по настройке](https://docs.easyshell.ai/configuration/bot-channels/)):

## Быстрый старт

```bash
git clone https://github.com/easyshell-ai/easyshell.git
cd easyshell
cp .env.example .env      # Отредактируйте .env при необходимости
docker compose up -d
```

Локальная сборка не требуется — готовые образы автоматически загружаются из [Docker Hub](https://hub.docker.com/u/laolupaojiao).

Откройте `http://localhost:18880` → войдите с логином `easyshell` / `easyshell@changeme`.

> **Хотите использовать GHCR?** Укажите в `.env`:
> ```
> EASYSHELL_SERVER_IMAGE=ghcr.io/easyshell-ai/easyshell/easyshell-server:latest
> EASYSHELL_WEB_IMAGE=ghcr.io/easyshell-ai/easyshell/easyshell-web:latest
> ```

> **Разработчик? Сборка из исходного кода:**
> ```bash
> docker compose -f docker-compose.build.yml up -d
> ```

## Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Server | Java 17, Spring Boot 3.5, Gradle, JPA/Hibernate, Spring AI, Spring Security |

## Структура проекта

```
easyshell/
├── easyshell-server/           # Центральный сервер управления (Java / Spring Boot)
├── easyshell-agent/            # Клиентский агент (Go, единый бинарный файл)
├── easyshell-web/              # Веб-интерфейс (React + Ant Design)
├── docker-compose.yml          # Развертывание в продакшене (готовые образы)
├── docker-compose.build.yml    # Разработка (локальная сборка из исходников)
├── Dockerfile.server           # Многоэтапная сборка Server + Agent
├── Dockerfile.web              # Многоэтапная сборка веб-интерфейса
├── .github/workflows/          # CI/CD: сборка и публикация образов Docker
└── .env.example                # Шаблон конфигурации окружения
```
